## Scope

This package supports the visual revision plan and execution records for the
current nine-slide North Slope gas hydrate ML deck. It collects public-safe
source links, reusable visual assets, icon ideas, Gmail-provided visual
references, and detailed prompts or execution notes.

well-log rows, restricted identifiers, populated runtime settings, trained
models, or derived sensitive outputs belong in the public deck.

### Slide 2: Introduction and Gas Hydrate Visuals

Goal: replace the old structural explorer image, add a visual gas hydrate
explanation, and give every point a visual.

Changes:

- Replace the old structural explorer image with
  `images/project_streamlit_structural_explorer_v2.png`.
- Add a gas hydrate visual using either the USGS hydrate-crystal image or a
  generated/source-backed methane hydrate cage diagram.
- Use four visual tiles instead of text-heavy bullets:
  - `What`: methane trapped in ice-like water cages.
  - `Where`: beneath Arctic permafrost and marine sediments, with North Slope

  - `Why`: large assessed North Slope resource and research value.
  - `Goal`: predict occurrence and saturation separately from reservoir quality
    and producibility.

  proof.

Recommended visuals:

- Top-left methane hydrate cage or USGS hydrate-crystal crop.
- Bottom-left four small visual tiles with one short phrase each.

### Slide 3: Parameters With Larger Visuals

Goal: reduce packed text and make parameter symbols reusable across the deck.

Changes:

- Replace paragraph-style parameter cards with larger icon tiles.
- Use 6 to 8 grouped tiles rather than 10 dense rows:
  - Lithology: `GR`.
  - Electrical: `Rt`.
  - Porosity/density: `RHOB`, `phi_D`.
  - NMR/fluid: `phi_NMR`, locked `S_h` or `NMR_SAT` target badge.
  - Elastic: `Vp`, `Vs`, `Vp/Vs`, `AI`.
  - Borehole QC: `CAL/DCAL`.
  - Context/calibration: `z / P-T`, `Core`.
- Each tile should show:
  - symbol and plain name;
  - one icon or mini-scene;
  - one short measurement phrase;
  - one caveat phrase.
- Move detailed measurement/caveat wording into speaker notes or backup text,
  not the slide body.

Recommended icon rule:

- Use the icons in `icon_registry.csv` consistently on slides 3, 4, 5, and 6.
- Make `S_h`, `Sgh`, and `NMR_SAT` visibly locked targets, not input features.

### Slide 4: Detailed ML Architecture

Goal: make the architecture detailed and readable, with each parameter linked
to gates, equations, branches, and outputs.

Changes:

- Start with `Approved logs` as icon chips, not a plain list:
  `GR`, `Rt`, `RHOB`, `phi_D`, `phi_NMR`, `Vp`, `Vs`, `CAL/DCAL`, `Core`, `z`.
- Give every QC gate its own box and line:
  - unit/provenance check;
  - depth alignment;
  - caliper/washout;
  - missingness/outliers;
  - lithology/reservoir gate;
  - target-leakage barrier.
- Add explicit feature-equation boxes:
  - `Vsh` from `GR`;
  - `phi_D` from `RHOB`;
  - `Delta_NMR-D = phi_D - phi_NMR`;
  - `Vp/Vs`;
  - `AI = rho_b * Vp`;
  - `mu-rho = rho_b * Vs^2`;
  - `lambda-rho = rho_b * (Vp^2 - 2Vs^2)`.
- Show model flow after equations:
  - train-only preprocessing;
  - complete-well split;
  - rule/logit/Ridge baseline;
  - tree/boosting models;
  - Keras ANN saturation test;
  - classification head;
  - saturation-regression head;
  - uncertainty/review flags;
  - predicted sweet-spot output.

Layout idea:

- Left vertical input stack.
- Middle QC-gate row and equation row.
- Right parallel model branches.
- Bottom red target-leakage rail separating `S_h`, `Sgh`, `NMR_SAT`, phase
  labels, and final rankings from input features.

### Slide 5: Parameter Behavior by Geologic Setting

Goal: replace abstract rectangles with visuals that show behavior in clean
sand, shale, hydrate, gas, and bad-hole conditions.

Changes:

- Use behavior panels instead of text-heavy rows.
- Suggested panels:
  - `Clean sand`: low `GR`, usable `phi`, no automatic hydrate label.
  - `Hydrate in sand`: high `Rt`, NMR-density separation, stiffer `Vp/Vs`
    behavior, clean reservoir gate.
  - `Shale`: high `GR`, bound water, poor reservoir quality.
  - `Free gas`: high `Rt` possible, `Vp` drops relative to `Vs`.
  - `Ice/cement/carbonate`: stiff and resistive mimic without hydrate proof.
  - `Bad hole`: caliper washout corrupts density, neutron/NMR, sonic, and
    resistivity.
- Each panel should use mini log curves, rock blocks, or crossplot regions.
- Limit text to the panel title plus one caveat line.

### Slide 6: Cleaner Geomechanical Feature Sketch

Goal: make the rock-physics logic clearer and less text-heavy.

Changes:

- Use one large flow: `Inputs -> Feature equations -> Interpretation checks`.
- Show P-wave and S-wave icons moving through a rock block.
- Make the equations large enough to read:
  - `AI = rho_b * Vp`
  - `Vp/Vs = Vp / Vs`
  - `mu-rho = rho_b * Vs^2`
  - `lambda-rho = rho_b * (Vp^2 - 2Vs^2)`
- Add a behavior comparison strip:
  - hydrate-supportive;
  - free-gas caveat;
  - shale caveat;
  - ice/cement/carbonate caveat;
  - overburden/compaction caveat;
  - bad-hole caveat.
- Keep the main takeaway visible: high resistivity or high velocity is evidence
  to review, not a hydrate label by itself.
